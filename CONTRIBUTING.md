## ğŸ§‘â€ğŸ’» Coding Conventions

### General
- Use **TypeScript** wherever possible
- Prefer clear, descriptive variable and function names
- Keep functions focused and small
- Avoid unnecessary complexity

### Frontend
- Follow React best practices (hooks, functional components)
- Use Tailwind utility classes consistently
- Keep UI components reusable and composable
- Use Framer Motion sparingly and meaningfully

### Backend
- Validate request inputs
- Handle errors gracefully
- Keep API responses consistent and typed
- Avoid hard-coding secrets or keys

### Database
- Use Drizzle ORM for all database interactions
- Avoid raw SQL unless necessary
- Keep migrations small and intentional
